# SRT (Systems-Roles-Table) для VK-offee

**Systems-Roles-Table** — метод организации знаний через таблицу 3×3 (9 семейств документов F1-F9 + метасемейство F0).

---

## 📊 Таблица 3×3

```
                     Предприниматель  Инженер           Менеджер
                     (Зачем?)         (Как устроено?)   (Как работает?)
┌────────────────────────────────────────────────────────────────────┐
│ Suprasystem        F1               F2                F3           │
│ (Надсистема)       Контекст         Окружение         Взаимодействие│
│                    и рынок          и интерфейсы      и партнёрства│
├────────────────────────────────────────────────────────────────────┤
│ System-of-Interest F4               F5                F6           │
│ (Целевая система)  Требования       Архитектура       Реализация   │
│                    и ценность       продукта          и процессы   │
├────────────────────────────────────────────────────────────────────┤
│ Constructor        F7               F8                F9           │
│ (Система создания) Принципы         Платформа         Команда      │
│                    и экономика      и инструменты     и методология│
└────────────────────────────────────────────────────────────────────┘
```

**F0** — метасемейство (управление хранилищем знаний)

---

## 🎯 Три системы для VK-offee

| Система | Что это для VK-offee |
|---------|---------------------|
| **Suprasystem (F1-F3)** | Рынок HoReCa, клиенты кофеен, партнёры |
| **System-of-Interest (F4-F6)** | Сеть кофеен VK-offee (продукт) |
| **Constructor (F7-F9)** | Команда менеджмента + инструменты |

---

## 📋 Семейства документов

| Семейство | Папка | Фокус |
|-----------|-------|-------|
| **F0** | `F0-Meta/` | Управление хранилищем (логика, отчёты, inbox) |
| **F1** | `F1-Suprasystem-Entrepreneur/` | Контекст и рынок HoReCa |
| **F2** | `F2-Suprasystem-Engineer/` | Окружение и интерфейсы с клиентами |
| **F3** | `F3-Suprasystem-Manager/` | Взаимодействие и партнёрства |
| **F4** | `F4-SoI-Entrepreneur/` | Требования и ценность кофеен |
| **F5** | `F5-SoI-Engineer/` | Архитектура сети кофеен |
| **F6** | `F6-SoI-Manager/` | Реализация и операционные процессы |
| **F7** | `F7-Constructor-Entrepreneur/` | Принципы и экономика управления |
| **F8** | `F8-Constructor-Engineer/` | Платформа и инструменты (боты, GitHub) |
| **F9** | `F9-Constructor-Manager/` | Команда и методология работы |

---

## 🔗 Связь с PACK

**PACK-cafe-operations** содержит формализованное доменное знание (методы, рецепты, стандарты).

**SRT** организует это знание по системам и ролям для удобной навигации.

**Правило:** Один документ может быть в нескольких семействах через симлинки или ссылки.

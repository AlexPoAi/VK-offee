# Инструкции для Claude Code и AI-агентов

Этот документ содержит инструкции для работы с базой знаний сети «Вкусный Кофе».

## Структура хранилища

База организована по методу **SRT (Systems-Roles-Table)** — таблица 3×3 систем и ролей:

- **F0** — Метасемейство (управление хранилищем)
- **F1-F3** — Надсистема (рынок HoReCa)
- **F4-F6** — Целевая система (сеть кофеен)
- **F7-F9** — Система создания (команда управления)

## Три системы проекта

### System-of-Interest (Целевая система)
**Сеть кофеен «Вкусный Кофе»** — то, что создаём.

### Suprasystem (Надсистема)
**Городская среда и рынок HoReCa** — система, в которую встроены кофейни.

### Constructor (Система создания)
**Команда управления** — система, которая создаёт и развивает сеть.

## Правила работы с хранилищем

### 1. Навигация
- Всегда начинай с [content/README.md](content/README.md)
- Используй таблицу SRT 3×3 для понимания структуры
- Следуй по ссылкам между документами

### 2. Терминология
- Используй термины из [glossary.md](content/0.Management/0.1.%20Логика%20хранилища%20и%20знаний/glossary.md)
- **P95** — 95-й перцентиль времени выдачи
- **AOV** — средний чек
- **CRR** — доля повторных визитов
- **NPS** — индекс лояльности
- **Фудкост** — себестоимость / выручка

### 3. Системное мышление
При анализе используй методы:
- **Чёрный ящик** — анализ входов/выходов без внутренних процессов
- **Прозрачный ящик** — анализ внутренних процессов
- **Холархия** — иерархия систем (надсистема → система → подсистемы)

### 4. Создание документов
При создании новых документов:
- Определи семейство (F0-F9)
- Используй YAML front matter:
  ```yaml
  ---
  title: "Название документа"
  section: "раздел"
  role: ["роль1", "роль2"]
  status: "green|yellow|red"
  owner: "Ответственный"
  last_reviewed: "YYYY-MM-DD"
  next_review: "YYYY-MM-DD"
  ---
  ```
- Добавь в [knowledge-inventory.md](content/0.Management/0.1.%20Логика%20хранилища%20и%20знаний/knowledge-inventory.md)

### 5. Обновление документов
- Проверяй актуальность ссылок
- Обновляй `last_reviewed` и `next_review`
- Меняй `status` при необходимости (green/yellow/red)

## Сессии стратегирования

### Что это такое
Сессия стратегирования — это глубокий анализ бизнеса сети кофеен с использованием системного мышления и методологии FPF (First Principles Framework). Цель — выявить проблемы, найти точки роста и предложить конкретные решения.

### Как запустить сессию
1. **Через Claude Code CLI:** Откройте терминал в папке `/Users/alexander/VK-offee` и запустите `claude`
2. **Через браузер (claude.ai):** Создайте новый проект, загрузите этот файл (CLAUDE.md) и скажите: "Запусти сессию стратегирования для VK-offee"

### Что делает агент в сессии стратегирования

#### 1. Изучение текущего состояния
Агент должен проанализировать:
- **Системное описание** ([system-description.md](content/2.VkusnyCoffeeNetwork/2.1.Meaning/system-description.md))
- **Бизнес-модель** ([business-model.md](content/3.ManagementTeam/3.1.Meaning/business-model.md))
- **Метрики и показатели** (из knowledge-base и финансовых документов)
- **Операционные процессы** ([operational-processes.md](content/2.VkusnyCoffeeNetwork/2.3.Operations/operational-processes.md))

#### 2. Анализ по трём системам (SRT)
Используй таблицу 3×3 для структурированного анализа:

**Надсистема (F1-F3):** Рынок HoReCa
- F1: Контекст — кто наши гости, какие сегменты, конкуренты
- F2: Окружение — районы, инфраструктура, трафик
- F3: Партнёрства — франчайзинг, поставщики, регуляторы

**Целевая система (F4-F6):** Сеть кофеен
- F4: Ценности — что мы обещаем гостям, миссия, видение
- F5: Архитектура — меню, продукты, рецептуры, стандарты
- F6: Операции — процессы, чек-листы, контроль качества

**Система создания (F7-F9):** Команда управления
- F7: Финансы — себестоимость, калькуляции, инвестиции
- F8: Инструменты — POS, KDS, поставщики, оборудование
- F9: Команда — обучение, роли, методология

#### 3. Ключевые вопросы для анализа
- Какие метрики показывают проблемы? (P95, AOV, CRR, NPS, Фудкост)
- Где узкие места в операциях?
- Какие процессы не документированы или устарели?
- Что мешает масштабированию?
- Какие возможности для роста выручки?
- Как улучшить гостевой опыт?
- Где можно снизить издержки без потери качества?

#### 4. Методы анализа
- **Чёрный ящик:** Анализ входов/выходов (гости приходят → получают опыт → возвращаются или нет)
- **Прозрачный ящик:** Анализ внутренних процессов (как готовятся напитки, как работает команда)
- **Холархия:** Понимание иерархии систем (кофейня → сеть → рынок HoReCa → городская среда)

#### 5. Ожидаемые результаты сессии
Агент должен подготовить:
1. **Диагностика:** Что работает хорошо, что плохо (используй светофор: 🟢🟡🔴)
2. **Проблемы:** Список выявленных проблем с приоритетами
3. **Гипотезы:** Предположения о причинах проблем
4. **Рекомендации:** Конкретные действия для улучшения
5. **План действий:** Что делать в первую очередь, что потом

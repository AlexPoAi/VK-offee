## 🎯 Роль этого репозитория в экосистеме

### VK-offee = Pack (Вторые принципы)

**Pack** — это **source-of-truth** для домена кофеен. Формализованное знание о том, как работает сеть «Вкусный Кофе».

**Иерархия принципов:**
- **Нулевые принципы (ZP)** — работают в любой области
- **Первые принципы (FPF)** — работают в любой дисциплине → `.fpf/FPF-Spec.md`
- **Вторые принципы (Pack)** — работают в конкретном домене → **этот репозиторий**
- **Третьи принципы (Downstream)** — работают в конкретном проекте → `creativ-convector`

### Связь с Downstream

**creativ-convector** — это Downstream/Governance репозиторий:
- Обработка входящих заметок
- Стратегирование
- Планы и координация
- Извлечение зна��ий для Pack

**Правило:** Downstream не может противоречить Pack. Если Pack обновился — Downstream обновляется вслед.

## 📚 Методология: FPF и SRT

### First Principles Framework (FPF)

**FPF** — "операционная система для мышления", трансдисциплинарная архитектура рассуждений.

**Ключевые принципы:**

| Принцип | Паттерн | Описание |
|---------|---------|----------|
| **Холонический фундамент** | A.1 | Всё — холон: одновременно целое и часть |
| **Ограниченный контекст** | A.1.1 | Значение локально, определено в границах контекста |
| **Строгое различение** | A.7 | Роль ≠ Метод ≠ Работа. Описание ≠ Объект |
| **F-G-R доверие** | B.3 | Доверие = (Формальность, Область, Надёжность) |
| **Цикл ADI** | B.5 | Абдукция → Дедукция → Индукция |
| **NQD-поиск** | C.18 | Новизна-Качество-Разнообразие |

### Systems-Roles-Table (SRT)

**SRT** — метод организации знаний через **таблицу 3×3** (9 семейств F1-F9 + метасемейство F0).

#### Таблица 3×3: Три системы × Три роли

```
                     Предприниматель  Инженер           Менеджер
                     (Зачем?)         (Как устроено?)   (Как работает?)
┌────────────────────────────────────────────────────────────────────┐
│ HoReCaMarket       F1               F2                F3           │
│ (Надсистема)       Контекст         Окружение         Партнёрства  │
│                    и рынок          и интерфейсы      и франшиза   │
├────────────────────────────────────────────────────────────────────┤
│ VkusnyCoffee       F4               F5                F6           │

│                    и описание       и архитектура     и качество   │
```

## 🔄 Workflow: Pack ↔ Downstream

### Поток знаний

```
Заметки (Telegram/Obsidian)
    ↓
creativ-convector (Downstream/Governance)
    ↓ обработка, стратегирование
    ↓ извлечение знаний
    ↓
VK-offee (Pack)
    ↓ формализация
    ↓
Агенты читают через MCP
    ↓
Действия (боты, автоматизация)
```

### Роли агентов

| Роль | Где работает | Что делает |
|------|--------------|------------|
| **Рецензент** | creativ-convector | Разбирает заметки, маршрутизирует |
| **Стратег** | creativ-convector | Формирует планы, создаёт отчёты |
| **Экстрактор** | creativ-convector → VK-offee | Извлекает знания для Pack |
| **Синхронизатор** | VK-offee ↔ creativ-convector | Следит за согласованностью |
| **Консультант** | VK-offee | Отвечает на вопросы из Pack |

## 📖 Правила работы с Pack

### 1. Навигация

- Всегда начинай с [content/README.md](content/README.md)
- Используй таблицу SRT 3×3 для понимания структуры
- Следуй по ссылкам между документами

### 2. Терминология

Используй термины из [glossary.md](content/0.Management/0.1.%20Логика%20хранилища%20и%20знаний/glossary.md):

- **P95** — 95-й перцентиль времени выдачи
- **AOV** — средний чек (Average Order Value)
- **CRR** — доля повторных визитов (Customer Retention Rate)
- **NPS** — индекс лояльности (Net Promoter Score)
- **Фудкост** — себестоимость / выручка

### 3. Системное мышление

При анализе используй методы:

- **Чёрный ящик** — анализ входов/выходов без внутренних процессов
- **Прозрачный ящик** — анализ внутренних процессов
- **Холархия** — иерархия систем (надсистема → система → подсистемы)

### 4. Создание документов

При создании новых документов:

1. **Определи семейство (F0-F9)**
2. **Используй YAML front matter:**
   ```yaml
   ---
   title: "Название документа"
   section: "раздел"
   role: ["роль1", "роль2"]
   status: "green|yellow|red"
   owner: "Ответственный"
   last_reviewed: "YYYY-MM-DD"
   next_review: "YYYY-MM-DD"
   fpf_principles: ["A.1", "B.3"]
   ---
   ```
3. **Добавь в knowledge-inventory.md**
4. **Проверь согласованность с FPF**

### 5. Обновление документов

- Проверяй актуальность ссылок
- Обновляй `last_reviewed` и `next_review`
- Меняй `status` при необходимости (green/yellow/red)
- **Важно:** При обновлении Pack уведоми Downstream (creativ-convector)

### Принципы

1. **Pack — source-of-truth**: Агенты читают знания из Pack, не придумывают
2. **Согласованность**: Downstream не противоречит Pack

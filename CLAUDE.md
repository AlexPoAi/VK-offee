# CLAUDE.md - Правила работы с VK-offee

> Инструкции для AI-агентов и разработчиков, работающих с репозиторием VK-offee

---

## 🎯 Миссия репозитория

VK-offee - это операционная база знаний сети кофеен «Вкусный Кофе». Репозиторий содержит все процессы, стандарты, рецепты и инструменты для управления кофейнями.

**Цель:** Обеспечить единый источник истины (Single Source of Truth) для всех операционных процессов.

---

## 📁 Архитектура репозитория

### Структура

```
VK-offee/
├── PACK-cafe-operations/      # Доменное знание (процессы, стандарты)
├── knowledge-base/            # Данные (CSV, таблицы, документы)
├── telegram-bot/              # Инструменты (бот для доступа к знаниям)
├── .github/scripts/           # Автоматизация (синхронизация)
├── MEMORY.md                  # Оперативная память проекта
├── WeekPlan.md               # Недельное планирование
└── CLAUDE.md                 # Этот файл
```

### Принципы организации

1. **PACK-cafe-operations** - единственный авторитетный источник процессов
2. **knowledge-base** - сырые данные из Google Sheets (автосинхронизация)
3. **telegram-bot** - downstream инструмент, читает из PACK и knowledge-base
4. **Не создавать дубликаты** - одна информация в одном месте

## 🔄 Протоколы работы

### Open (Начало сессии)

Перед началом работы:

1. **Прочитай MEMORY.md** - узнай текущий контекст
2. **Проверь WeekPlan.md** - есть ли задача в плане?
3. **Объяви роль и цель:**
   ```
   Роль: [Операционный менеджер / Разработчик / Экстрактор знаний]
   Задача: [Что делаем]
   Рабочий продукт: [Что создаём]
   Время: [Оценка]
   ```

### Work (Работа)

Во время работы:

1. **Фиксируй знания:**
   - Правило для всех → `CLAUDE.md` (этот файл)
   - Доменное знание → `PACK-cafe-operations/`
   - Данные → `knowledge-base/`
   - Урок → `MEMORY.md`

2. **Используй различения** из `PACK-cafe-operations/01-domain-contract/distinctions.md`

3. **Коммить часто** с понятными сообщениями:
   ```
   feat: Добавлен процесс приготовления капучино
   fix: Исправлена синхронизация Google Sheets
   docs: Обновлены различения в Pack
   ```

### Close (Завершение сессии)

После работы:

1. **Обнови MEMORY.md:**
   - Что сделано
   - Что узнал
   - Что осталось

2. **Обнови WeekPlan.md:**
   - Отметь выполненные задачи
   - Обнови статус

3. **Закоммить и запушить:**
   ```bash
   git add .
   git commit -m "session: [краткое описание]"
   git push
   ```

## 📋 Правила работы с файлами

### CSV файлы (knowledge-base/)

- **НЕ редактировать вручную** - они синхронизируются из Google Sheets
- **Проверять на HTML** - если файл содержит `<!DOCTYPE html>`, это ошибка
- **Использовать скрипты** из `.github/scripts/` для синхронизации

### Markdown файлы (PACK-cafe-operations/)

- **Использовать шаблоны** для карточек методов и различений
- **Версионировать** - указывать дату обновления внизу
- **Линковать** - использовать относительные ссылки между файлами

### Python скрипты

- **Документировать** - docstrings для всех функций
- **Логировать** - использовать `logging` вместо `print`
- **Обрабатывать ошибки** - try/except с понятными сообщениями

## 🤖 Работа с Telegram-ботом

### Архитектура бота

```
telegram-bot/
├── bot.py                    # Основной файл
├── .env                      # Токены (не коммитить!)
├── requirements.txt          # Зависимости
└── venv/                     # Виртуальное окружение
```

### Правила

1. **Бот читает, не пишет** - только read-доступ к knowledge-base
2. **Использует OpenAI API** - для генерации ответов
3. **Логирует всё** - для отладки поиска
4. **Работает локально** - пока не задеплоен

### Запуск бота

```bash
cd telegram-bot
source venv/bin/activate
python3 bot.py
```

## 🔍 Работа с базой знаний

### Поиск информации

1. **Сначала PACK** - ищи в структурированных процессах
2. **Потом knowledge-base** - если нужны сырые данные
3. **Используй Grep** - для поиска по содержимому:
   ```bash
   grep -r "капучино" knowledge-base/
   ```

### Добавление нового знания

**Если это процесс:**
1. Создай карточку в `PACK-cafe-operations/03-methods/`
2. Используй формат:
   ```markdown
   # ID: CAF.M.XXX
   # Название: [Процесс]
   # Роль: [Кто выполняет]
   # Входные данные: [Что нужно]
   # Шаги: [Как делать]
   # Выходной продукт: [Результат]
   # Failure modes: [Типичные ошибки]
   ```

**Если это данные:**
1. Добавь в Google Sheets
2. Запусти синхронизацию:
   ```bash
   cd .github/scripts
   python3 sync-google-sheets.py
   ```

## 🚨 Типичные ошибки и решения

### Ошибка: CSV файл содержит HTML

**Причина:** Файл скачан через прямую ссылку без авторизации

**Решение:**
```bash
cd .github/scripts
python3 fix_main_sheet.py
```

### Ошибка: Бот не находит данные

**Причина:** Проблема с поиском или передачей контекста

**Отладка:**
1. Проверь логи: `tail -f telegram-bot/bot.log`
2. Проверь, что файл существует: `ls knowledge-base/`
3. Проверь содержимое: `head knowledge-base/файл.csv`

### Ошибка: Git конфликт при push

**Решение:**
```bash
git pull --rebase
# Разреши конфликты
git add .
git rebase --continue
git push
```

## 📊 Метрики качества

### Для PACK-cafe-operations

- ✅ Все процессы имеют ID
- ✅ Все различения имеют примеры ошибок

- ✅ Время ответа < 5 секунд

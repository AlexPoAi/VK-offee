# DOMAIN-CONTRACT
DOMAIN: PACK-kitchen / ТИП: Блюдо

---

## 1. ОПРЕДЕЛЕНИЕ
Блюдо — это целевая система PACK-kitchen: физическая пищевая композиция, возникающая в момент завершения сборки и предназначенная для передачи на бар VK-offee.

---

## 2. СОСТАВ

**Базовый (всегда):**
- пищевое вещество (композиция из ингредиентов и/или заготовок)

**Расширенный (в целевом состоянии):**
- пищевое вещество
- упаковка (подсистема)
- маркировка (подсистема)

> Упаковка и маркировка — подсистемы Блюда, а не отдельные объекты. Они появляются при переходе в соответствующие состояния.

---

## 3. СВЯЗИ

- создаётся → повар
- создаётся из → ингредиенты / заготовки
- производится в → кухонный цех
- передаётся в → PACK-cafe-operations (как Порция еды)

---

## 4. ХАРАКТЕРИСТИКИ

- наименование (по меню)
- состав (из каких ингредиентов/заготовок собрано)
- масса / объём / количество порций
- дата и время изготовления
- срок годности
- температура подачи / хранения до передачи
- наличие упаковки и маркировки

---

## 5. СОСТОЯНИЯ

собрано → остывает / стабилизируется → упаковывается → упаковано → маркируется → маркировано → готово к передаче → передано (прекращает существование в PACK-kitchen)

---

## 6. ПРАВИЛА

- возникает в момент завершения физической сборки (не после упаковки, не после маркировки)
- упаковка и маркировка — изменения состояния, не создание новой системы
- должно быть упаковано и промаркировано до передачи
- передаётся только в состоянии «готово к передаче»
- при передаче прекращает существование в PACK-kitchen
- рецепт = U.MethodDescription (описание метода), не часть системы Блюдо

---

## 7. ПРИМЕР

Блюдо «Сэндвич с курицей»:
- собрано из заготовки «варёное куриное филе» + свежие ингредиенты
- собрано 28.03.2026 в 11:30 → Блюдо возникло
- упаковано в контейнер, промаркировано (дата, состав, срок)
- передано на бар в 11:45 → в PACK-kitchen прекратило существование
- в PACK-cafe-operations появилось как «Порция еды: Сэндвич с курицей»

---

## 8. ПРОВЕРКА РАЗЛИЧЕНИЙ (A.7)

- Блюдо (PACK-kitchen) ≠ Порция еды (PACK-cafe-operations): один физический объект, два BoundedContext — не нарушение A.7, смена контекста при передаче
- Блюдо ≠ Рецепт: рецепт = U.MethodDescription (знание «как делать»), Блюдо = U.System (физический результат)
- Блюдо ≠ Заготовка: заготовка — промежуточный ресурс (вход), Блюдо — целевая система (выход)
- Упаковка/маркировка ≠ новая система: это изменения состояния одной системы

✔ различения соблюдены

## FPF-основание

- Блюдо = целевая система PACK-kitchen (A.1: то, ради чего существует домен)
- Возникает в момент физической сборки (A.4: появление системы при объединении частей)
- Прекращает существование при передаче — смена BoundedContext, не уничтожение физики (A.1.1)
- Цепочка: Ингредиент → [обработка] → Заготовка → [сборка] → Блюдо → [передача] → Порция еды (PACK-cafe-operations)

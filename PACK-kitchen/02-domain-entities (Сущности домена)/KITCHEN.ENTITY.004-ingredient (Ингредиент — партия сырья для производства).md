# DOMAIN-CONTRACT
DOMAIN: PACK-kitchen / ТИП: Ингредиент

---

## 1. ОПРЕДЕЛЕНИЕ
Ингредиент — это партия пищевого сырья, принятая кухней и предназначенная для переработки в заготовку или прямого включения в блюдо.

---

## 2. СОСТАВ

- вещество (сырьё)
- упаковка поставщика
- маркировка приёмки (дата, фактическая масса)

> ❌ накладная/сертификат — это описание (U.Episteme), не физическая часть системы

---

## 3. СВЯЗИ

- поступает от → поставщик
- принимается → повар
- хранится в → кухонный цех
- преобразуется в → заготовка
- используется напрямую в → блюдо

---

## 4. ХАРАКТЕРИСТИКИ

- наименование
- масса / объём (может изменяться при переходах состояний)
- дата поступления
- срок годности
- температурный режим хранения
- текущее физическое состояние (заморожено / охлаждено / разморожено)
- органолептическое состояние при приёмке

---

## 5. СОСТОЯНИЯ

принят → хранится (заморожен / охлаждён) → размораживается → разморожен → в работе → полностью использован (прекращает существование)

---

## 6. ПРАВИЛА

- принимается по массе и органолептике → несоответствие = отказ
- хранится с соблюдением температурного режима
- используется по принципу FIFO
- не используется после истечения срока годности
- при переходе состояния (например: заморожен → разморожен) → должна обновляться маркировка
- при изменении состояния → могут изменяться характеристики (например: масса после разморозки)
- используется только как вход → заготовка или → блюдо
- при преобразовании в заготовку → ингредиент прекращает существование, создаётся новая система (заготовка) с новой маркировкой
- запрещено смешивать партии с разной маркировкой (например: старое варёное филе ≠ новая партия сырого филе)

---

## 7. ПРИМЕР

Ингредиент «куриное филе» (партия 5 кг),
дата поступления 28.03.2026,
хранится в замороженном виде.

29.03.2026 разморожено → масса изменилась до 4.7 кг, обновлена маркировка.
Далее используется для создания заготовки «варёное куриное филе»,
которая получает новую маркировку и прекращает существование как ингредиент.

---

## 8. ПРОВЕРКА РАЗЛИЧЕНИЙ (A.7)

- Ингредиент ≠ Заготовка (сырьё vs обработанный продукт)
- Ингредиент ≠ Остаток (физическая партия vs учёт в PACK-cafe-operations)
- Изменение состояния ≠ новая система (пока не произошло преобразование)
- Преобразование → создаёт новую систему (заготовка)

✔ различения соблюдены

---

## FPF-основание

- Цепочка преобразования: Ингредиент → [обработка] → Заготовка → [сборка] → Блюдо
- При включении в следующую систему — предыдущая прекращает существовать (A.4)
- Ингредиент в PACK-kitchen ≠ ингредиент в PACK-bar: разные BoundedContext, разные роли объекта (A.1.1)
- Документы (накладная) = U.Episteme, не входят в состав физической системы (A.7)

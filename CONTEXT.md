После КАЖДОГО изменения файлов:
1. `git add .`
2. `git commit -m "описание"`
3. **`git push origin main`** ← ОБЯЗАТЕЛЬНО!

**Почему это критично:**
- Без `git push` изменения остаются только локально
- Веб-агенты Claude.ai НЕ ВИДЯТ локальные изменения
- Другие агенты НЕ ВИДЯТ локальные изменения
- Только я (Claude Code) вижу локальные файлы

**Что установлено:**
- ✅ Git Hook: автоматический push после каждого commit
- ✅ Cron: автоматическая синхронизация Google Drive → GitHub (9:00 ежедневно)
- ✅ Cron: автоматическая синхронизация Google Sheets → GitHub (9:10 ежедневно)
- ✅ auto-sync.sh: автоматический commit+push после синхронизации

**Проверка:** После работы выполнить `git log origin/main..HEAD`
- Если показывает коммиты → они НЕ в GitHub → выполнить `git push`
- Если пусто → всё в GitHub ✅

## 🏗️ АРХИТЕКТУРА: VK-offee = Pack

### Роль этого репозитория

**VK-offee = Pack (Вторые принципы)**

- Source-of-truth для домена кофеен «Вкусный Кофе»
- Формализованное знание (content/ + knowledge-base/)
- Структура SRT (F0-F9)
- База знаний для Telegram-бота и AI-агентов

### Связь с Downstream

**creativ-convector = Downstream/Governance (Третьи принципы)**
- Обработка входящих заметок
- Стратегирование и анализ
- Извлечение знаний для Pack

**Правило:** Downstream не может противоречить Pack. Pack — источник истины.

### Иерархия принципов

```
Нулевые принципы (ZP) — работают в любой области
    ↓
Первые принципы (FPF) — работают в любой дисциплине
    ↓ .fpf/FPF-Spec.md
Вторые принципы (Pack) — работают в конкретном домене
    ↓ VK-offee (ТЫ ЗДЕСЬ)
Третьи принципы (Downstream) — работают в конкретном проекте
    ↓ creativ-convector
```

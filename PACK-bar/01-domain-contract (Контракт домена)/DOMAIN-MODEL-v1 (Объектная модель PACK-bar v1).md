# Объектная модель PACK-bar

## Объекты

### [тип] напиток
```
напиток
  ├── название
  ├── объём        ← характеристика (30мл / 60мл — не новый рецепт)
  ├── температура  ← характеристика
  └── категория    ← характеристика (кофе / не кофе / альтернатива)
```

### [тип] рецепт
```
рецепт
  ├── напиток              → [напиток]
  ├── время приготовления  ← характеристика
  ├── температура          ← характеристика
  ├── оборудование         → [оборудование]
  └── ингредиенты [1..n]
        └── строка рецепта
              ├── ингредиент → [ингредиент]
              └── количество ← живёт здесь (18г / 150мл)
```
1 рецепт → 1 базовый напиток. Вариации (объём, температура) = характеристики, не новый рецепт.

**Пример: капучино**
```
рецепт: капучино
  ├── напиток: капучино
  ├── время: 60 сек
  ├── температура: 65°C
  ├── оборудование: кофемашина
  └── ингредиенты:
        ├── кофе   — 18 г
        └── молоко — 150 мл
```

### [тип] ингредиент
```
ингредиент
  ├── название
  └── единица измерения (г / мл / шт)
      (количество — только в контексте рецепта)
```

### [тип] оборудование
```
оборудование
  └── название типа (кофемашина / кофемолка / питчер...)
```
Типы → PACK-bar. Экземпляры на конкретной точке → PACK-cafe-operations.

## Связи
```
рецепт ──────────────► напиток
                          │
ингредиенты [1..n] ──────┤ (через строки рецепта)
                          │
оборудование ────────────┘

напиток → передаётся → в заказ (PACK-cafe-operations)
```

## Методы (не объекты)
- **приготовление напитка** → использует рецепт как инструкцию (03-methods)

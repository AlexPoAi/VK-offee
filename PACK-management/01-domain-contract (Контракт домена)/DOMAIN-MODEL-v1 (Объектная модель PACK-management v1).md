# Объектная модель PACK-management

## Объекты

### [тип] смена
```
смена
  ├── кофейная точка / кухонный цех
  ├── время начала / окончания  ← A.17
  ├── сотрудники [1..n]         ← ссылка PACK-hr
  ├── заказы [1..n]             (барная смена)
  ├── оплаты [1..n]             (барная смена → выручка)
  ├── заказы кухни [1..n]       (кухонная смена)
  └── поставки [1..n]           (кухонная смена)
```

### [тип] период учёта
```
период учёта
  ├── начало / конец            (~2 недели)
  ├── сотрудники [список]       ← формируется через смены
  │     правило: ≥1 смена в периоде → в списке
  ├── включает → заказы
  ├── включает → поставки
  └── заканчивается → инвентаризация (документ)
```

### [тип] инвентаризация (документ)
```
инвентаризация
  ├── дата
  ├── точка           → [кофейная точка]
  └── позиции [1..n]
        └── позиция
              ├── объект       → [ингредиент] | [порция еды]
              ├── факт         (реальный остаток)
              ├── расчёт       (по системе)
              └── расхождение  = факт − расчёт

Пример:
  молоко: факт 8л / расчёт 10л / расхождение −2л
```

### [тип] возмещение
```
возмещение
  ├── период          → [период учёта]
  ├── общая сумма     (= сумма недостачи)
  └── сотрудники [1..n]
        └── доля = общая сумма / кол-во сотрудников

Правило: равномерное распределение, без индивидуальной вины
Пример: 6000₽ / 3 сотрудника = 2000₽ каждому
```

## Метрики (вычисления, не объекты)
```
потери (грязные) = сумма недостачи
потери (чистые)  = недостача − возмещение
точность учёта   = факт / расчёт × 100%
% потерь         = недостача / оборот × 100%

Источники: заказы + поставки + инвентаризация
```

## Методы (не объекты) → 03-methods
- **проведение инвентаризации** — как делать (регламент)

## Разделение данные / выводы
```
PACK-cafe-operations     → факты (заказы, поставки, остатки, расхождения)
PACK-management          → учёт и ответственность (период, возмещение, метрики)
```

## Цепочка ответственности
```
смены → список сотрудников периода
расхождение (cafe) → возмещение (management) → доли сотрудникам
```

## 🎯 ФАЗА 1: Анализ и Подготовка (1-2 недели)

### 1.1 Извлечение доменов из knowledge-base

**Задача:** Идентифицировать и каталогизировать все предметные области

**Действия:**
```
knowledge-base/
├─ БАР/              → Домен: Барная стойка
├─ КУХНЯ/            → Домен: Кухня
├─ ДЛЯ ШЕФА/         → Домен: Управление кухней
└─ ...
```

**Результат:** Список всех доменов с описанием границ (Bounded Contexts)

**Выход:** `DOMAINS-CATALOG.md`

### 1.2 Определение границ каждого домена

**Для каждого домена:**
1. Определить ограниченный контекст (Bounded Context)
2. Определить сущности домена (Domain Entities)
3. Выделить различения (Distinctions)
4. Создать онтологию (Ontology)

**Шаблон:** `PACK-cafe-operations/01-domain-contract/01A-bounded-context.md`

**Результат:**
- Файлы `01A-bounded-context.md` для каждого домена
- Файлы `01B-distinctions.md` для каждого домена
- Файлы `01C-ontology.md` для каждого домена

## 🎯 ФАЗА 2: Разработка методов (2-3 недели)

### 2.1 Извлечение методов из knowledge-base

**Для каждого домена:**
1. Найти все процессы/методы в knowledge-base
2. Применить `process/07-method-and-product-extraction.md`
3. Создать карточку метода

**Структура метода:**
```
CAFE.METHOD.XXX-название-метода.md

ID: CAFE.METHOD.XXX
Name: Полное название метода
Description: Описание
Inputs: Входные данные
Outputs: Выходные данные
Distinctions: Ссылки на различения
Work Products: Ссылки на рабочие продукты
```

**Результат:** Файлы в `PACK-cafe-operations/03-methods/`

### 2.2 Обновление индекса методов

**Файл:** `PACK-cafe-operations/02-domain-entities/02C-methods-index.md`

**Содержит:** Список всех методов по доменам

## 🎯 ФАЗА 3: Разработка рабочих продуктов (2 недели)

Name: Полное название

## 📋 ПРОЦЕСС РАЗРАБОТКИ КАЖДОГО ДОМЕНА

### Для каждого домена следовать SPF (12 стадий):

```
00. Process Overview        → Понимание процесса создания знаний
01. Domain Selection        → Выбран домен (БАР, КУХНЯ, и т.д.)
02. Bounded Context         → Определены границы домена
03. Distinctions Work       → Выделены различения
04. Domain Entities         → Идентифицированы сущности
05. Information Ingestion   → Собрана информация из knowledge-base
06. Analysis & Formalization → Анализ и формализация
07. Method & Product Extraction → Извлечение методов и WP
08. Failure Modes Extraction → Извлечение ошибок
09. SoTA Annotation         → SoTA статус
10. Map Maintenance         → Обновление карт
11. Review & Evolution      → Цикл пересмотра
```

**Для каждой стадии:**
1. Прочитать файл `process/XX-название-стадии.md`
2. Следовать инструкциям
3. Создать required work products
4. Запустить `process-lint` перед коммитом

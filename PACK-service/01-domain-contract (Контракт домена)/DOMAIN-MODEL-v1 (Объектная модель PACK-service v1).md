# Объектная модель PACK-service

## Объекты

### [тип] стол
```
стол
  ├── номер
  ├── вместимость
  ├── зона (зал / терраса / улица)
  └── посадки [0..n]
```
Тип → PACK-service. Экземпляры на конкретной точке → PACK-cafe-operations.

### [тип] посадка
```
посадка
  ├── стол              → [стол]
  ├── время начала      ← характеристика (A.17)
  ├── время окончания   ← характеристика (A.17)
  └── заказы [0..n]     ← каждый = отдельный чек
        └── заказ → [заказ] (PACK-cafe-operations)
```

Производные метрики:
- длительность посадки = окончание − начало
- оборот стола = последовательность посадок

**Пример:**
```
стол №1 / посадка 10:00–10:30
  ├── заказ #1 → кофе → оплата картой
  └── заказ #2 → еда  → оплата наличными
```

## Правила

- **посадка = группировка** заказов за столом, не объединение чеков
- **финансы остаются на уровне заказа** (оплата в PACK-cafe)
- **заказ независим от стола**: takeaway → без посадки ✅
- **клиент = гость** — один тип, две роли (A.7 соблюдён)

## Связи
```
стол [1] → посадки [0..n] → заказы [0..n] → оплаты [1..n]
```

## Не объекты
- **обслуживание** → метод (03-methods)
- **официант** → ссылка из PACK-hr ([сотрудник])

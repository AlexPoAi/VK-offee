## 🎯 Текущая ситуация

### Две базы знаний в VK-offee

| База | Статус | Проблемы |
|------|--------|----------|
| **content/** | ✅ Хорошо | Структура SRT (F0-F9), YAML frontmatter, UTF-8 |
| **knowledge-base/** | ⚠️ Требует стандартизации | Неправильная кодировка, нет frontmatter, старая структура |

### Проблемы в knowledge-base/

1. **Кодировка:** Файлы в us-ascii вместо utf-8 (кракозябры при чтен��и)
2. **Нет frontmatter:** Нет метаданных (status, role, owner)
3. **Старая структура:** Папки на русском, нет привязки к F0-F9
4. **Дубликаты:** Много отчётов синхронизации

## 🔄 Что нужно сделать

### Вариант 1: Миграция knowledge-base/ → content/

**Идея:** Перенести все файлы из `knowledge-base/` в `content/` по структуре SRT.

**Плюсы:**
- Единая структура Pack
- Все документы с frontmatter
- Правильная кодировка
- Привязка к F0-F9

**Минусы:**
- Большая работа (много файлов)
- Нужно определить семейство для каждого документа
- Может сломать существующие ссылки

**Оценка:** 📅 2-3 дня работы

### Вариант 2: Стандартизация knowledge-base/ на месте

**Идея:** Оставить `knowledge-base/` как есть, но:
1. Исправить кодировку всех файлов
2. Добавить frontmatter
3. Создать индекс с привязкой к F0-F9

**Плюсы:**
- Меньше работы
- Не ломает существующие ссылки
- Можно делать постепенно

**Минусы:**
- Две базы знаний (content/ и knowledge-base/)
- Сложнее поддерживать

**Оценка:** 📅 1 день работы

### Вариант 3: Гибридный подход (рекомендуется)

**Идея:**
1. **content/** = Pack (формализованное знание) — уже готово ✅
2. **knowledge-base/** = Рабочие документы (Google Drive синхронизация) — исправить кодировку
3. Telegram-бот ищет в обеих базах

**Что делаем:**

#### Шаг 1: Исправить кодировку в knowledge-base/

```bash
# Скрипт для исправления кодировки
find knowledge-base/ -name "*.md" -exec sh -c '
  iconv -f WINDOWS-1251 -t UTF-8 "$1" > "$1.tmp" && mv "$1.tmp" "$1"
' _ {} \;
```

#### Шаг 2: Добавить минимальный frontmatter

Для каждого файла в `knowledge-base/` добавить:

```yaml
---
source: "Google Drive"
type: "рабочий документ"
---
```

#### Шаг 3: Создать индекс knowledge-base/

Файл `knowledge-base/INDEX.md` с привязкой к F0-F9:

```markdown
# Индекс knowledge-base

## F4-F6: Целевая система (Операции)
- БАР/ → F6 (Операции и качество)
- КУХНЯ/ → F6 (Операции и качество)

## F7-F9: Система создания
- Персонал/ → F9 (Команда и обучение)
- Документы/ → F7 (Финансы и принципы)
- ПОСТАВЩИКИ/ → F8 (Поставщики и инструменты)
```

#### Шаг 4: Обновить Telegram-бота

Бот ищет в обеих базах:
1. `content/` — формализованное знание (Pack)
2. `knowledge-base/` — рабочие документы

**Плюсы:**
- Минимум работы
- Не ломает существующую структуру
- Бот работает с обеими базами
- Можно делать постепенно

**Минусы:**
- Две базы знаний (но это нормально)

**Оценка:** 📅 2-3 часа работы

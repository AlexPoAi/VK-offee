# PROCESS-LINT - Проверка перед коммитом

Обязательная проверка для всех AI-агентов перед коммитом изменений в Pack.

---

## ⚠️ КРИТИЧЕСКИ ВАЖНО

**Process-lint НЕ опционален.** Каждое изменение в `PACK-cafe-operations/` должно пройти эту проверку.

---

## 🔍 Универсальные проверки (для всех изменений)

### ❌ Запрещённый язык (Didactic Language)

Проверь, что НЕТ этих слов:
- "step", "step 1", "step 2"
- "lesson", "module", "week"
- "try this", "exercise"
- "first/then", "in N days"
- "implement", "practice"

**Почему:** Pack фиксирует ЧТО существует, не КАК учить.

**Как исправить:** Переформулируй без дидактики.

❌ **Плохо:**
```markdown
Step 1: Heat the milk
Step 2: Pour into cup
Try this exercise to practice
```

✅ **Хорошо:**
```markdown
## Процесс
1. Нагреть молоко до 60-65°C
2. Влить в чашку
```

## 📋 Проверки для Method

### ✅ Обязательные элементы:

- [ ] Есть ID: `CAFE.METHOD.<NNN>`
- [ ] Есть секция "Назначение"
- [ ] Есть секция "Входы"
- [ ] Есть секция "Процесс"
- [ ] Есть секция "Выходы"
- [ ] Есть ссылка на Work Product (HG-1)
- [ ] Есть ссылка на Distinction (HG-2)
- [ ] Метод НЕ сценарий (HG-7)
- [ ] Нет дидактического языка (HG-6)

### ❌ Типовые ошибки:

**Ошибка 1: Сценарий вместо метода**

❌ **Плохо:**
```markdown
Вчера Иван готовил капучино и сделал так...
```

✅ **Хорошо:**
```markdown
## Процесс приготовления капучино
1. Приготовить эспрессо
2. Взбить молоко
3. Влить молоко в эспрессо
```

**Ошибка 2: Нет ссылки на WP**

❌ **Плохо:**
```markdown
## Выходы
Готовый капучино
```

✅ **Хорошо:**
```markdown
## Выходы
- [CAFE.WP.002 - Капучино](../04-work-products/CAFE.WP.002-cappuccino.md)
```

## 📦 Проверки для Work Product

### ✅ Обязательные элементы:

- [ ] Есть ID: `CAFE.WP.<NNN>`
- [ ] Есть секция "Описание"
- [ ] Есть секция "Observability criteria" (HG-3)
- [ ] Есть ссылка на Method
- [ ] Критерии наблюдаемости конкретны

### ❌ Типовые ошибки:

**Ошибка 1: Нет observability criteria**

❌ **Плохо:**
```markdown
## Описание
Капучино — это кофейный напиток
```

✅ **Хорошо:**
```markdown
## Observability criteria
- Объём: 180 мл
- Температура: 60-65°C
- Соотношение эспрессо:молоко:пена = 1:1:1
- Визуально: три слоя различимы
```

**Ошибка 2: Критерии неконкретны**

❌ **Плохо:**
```markdown
## Observability criteria
- Вкусный
- Красивый
```

✅ **Хорошо:**
```markdown
## Observability criteria
- Температура: 60-65°C (измеряется термометром)
- Пена: высота 1-1.5 см (измеряется линейкой)
```

## ⚠️ Проверки для Failure Mode

### ✅ Обязательные элементы:

- [ ] Есть ID: `CAFE.FAIL.<NNN>`
- [ ] Есть секция "Описание"
- [ ] Есть секция "Detection test" (HG-4)
- [ ] Есть ссылка на Method
- [ ] Есть ссылка на Distinction
- [ ] Тест обнаружения конкретен

### ❌ Типовые ошибки:

**Ошибка 1: Нет detection test**

❌ **Плохо:**
```markdown
## Описание
Молоко перегрето
```

✅ **Хорошо:**
```markdown
## Detection test
1. Измерить температуру молока термометром
2. Если > 70°C → перегрето
3. Визуально: пузыри на поверхности
4. Вкус: горький привкус
```

## 🗺️ Проверки для структурных изменений

### ✅ Обязательные действия:

- [ ] Обновлён индекс `02C-methods-index.md` (если добавлен Method)
- [ ] Обновлена карта в `07-map/` (HG-5)
- [ ] Обновлён README в соответствующей папке SRT/

### ❌ Типовая ошибка:

**Ошибка: Добавлен Method, но не обновлена карта**

❌ **Плохо:**
```bash
git add PACK-cafe-operations/03-methods/CAFE.METHOD.003-latte.md
git commit
# Карта не обновлена!
```

✅ **Хорошо:**
```bash
git add PACK-cafe-operations/03-methods/CAFE.METHOD.003-latte.md
git add PACK-cafe-operations/07-map/CAFE.MAP.001.md
git commit
```

## 🚦 Hard Gates (блокируют коммит)

| Gate | Проверка | Блокирует |
|------|----------|-----------|
| **HG-1** | Method без WP link | ✋ STOP |
| **HG-2** | Method без distinction link | ✋ STOP |
| **HG-3** | WP без observability criteria | ✋ STOP |
| **HG-4** | FM без detection test | ✋ STOP |
| **HG-5** | Structural change без map update | ✋ STOP |
| **HG-6** | Didactic language | ✋ STOP |
| **HG-7** | Scenario вместо method | ✋ STOP |
| **HG-8** | SoTA без revision criterion | ✋ STOP |

## 🔧 Как запустить process-lint

### Автоматически (рекомендуется):

```bash
# В будущем будет скрипт
./scripts/process-lint.sh
```

### Вручную:

1. Открой `process/process-lint.md`
2. Пройдись по всем проверкам
3. Исправь ошибки
4. Повтори

# 🔧 Настройка MCP сервера для доступа к VK-offee

**Проблема:** Агенты Claude (веб-версия) не могут найти файлы в VK-offee, потому что у них нет доступа к локальному репозиторию.

**Решение:** Настроить MCP (Model Context Protocol) сервер для доступа к базе знаний.

---

## 🎯 Три способа дать доступ агентам

### Способ 1: MCP Filesystem Server (локально) ✅ Рекомендуется

**Для кого:** Claude Desktop, Claude Code (локальные агенты)

**Как настроить:**

1. **Создать конфигурацию MCP:**

```bash
# Путь к конфигу Claude Desktop
nano ~/Library/Application\ Support/Claude/claude_desktop_config.json
```

2. **Добавить VK-offee сервер:**

```json
{
  "mcpServers": {
    "vk-offee-pack": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/alexander/VK-offee"
      ]
    },
    "creativ-convector-downstream": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/alexander/Documents/creativ-convector.nocloud"
      ]
    }
  }
}
```

3. **Перезапустить Claude Desktop**

4. **Проверить подключение:**
   - Открыть Claude Desktop
   - Спросить: "Найди файл Калькулятор для подсчета приготовленных продуктов в VK-offee"
   - Агент должен найти файл через MCP

**Результат:** Локальные агенты Claude смогут читать файлы из VK-offee и creativ-convector.

---

### Способ 2: GitHub как источник (для веб-агентов) ⏳

**Для кого:** Claude.ai (веб-версия), другие облачные агенты

**Проблема:** Веб-агенты не имеют доступа к локальным файлам.

**Решение:** Использовать GitHub как источник данных.

**Как работает:**

```
Веб-агент Claude → GitHub API → VK-offee репозиторий → Файлы
```

**Настройка:**

1. **Убедиться, что всё закоммичено в GitHub** ✅ (уже сделано)

2. **Дать агенту правильную инструкцию:**

```
Найди файл "Калькулятор для подсчета приготовленных продуктов.csv"
в репозитории https://github.com/alexpoaiagent-sudo/VK-offee
в папке knowledge-base/КУХНЯ/
```

3. **Или использовать GitHub API:**

```python
import requests

url = "https://api.github.com/repos/alexpoaiagent-sudo/VK-offee/contents/knowledge-base/КУХНЯ"
response = requests.get(url)
files = response.json()

# Найти файл
for file in files:
    if "Калькулятор" in file['name']:
        print(file['name'])
        # Скачать содержимое
        content_url = file['download_url']
        content = requests.get(content_url).text
        print(content)
```

**Ограничение:** Веб-агент не может напрямую читать файлы из GitHub без специальных инструкций.

### Способ 3: Telegram-бот (универсальный) ✅ Лучший для веб-агентов

**Для кого:** Любой пользователь (ты, менеджеры, сотрудники)

**Как работает:**

```
Ты → Telegram-бот → VK-offee (Pack) → AI-ответ
```

**Преимущества:**
- Работает из любого места
- Не нужен доступ к GitHub
- AI-ответы через GPT-4o-mini
- Поиск по всей базе знаний

**Запуск:**

```bash
cd ~/VK-offee/telegram-bot
./start.sh
```

**Использование:**

```
Ты пишешь боту в Telegram:
"Найди калькуляции кухни"

Бот отвечает:
"Найдено: Калькулятор для подсчета приготовленных продуктов.csv
Содержимое: [показывает данные]"
```

## 🔍 Почему веб-агент не нашёл файл?

### Причина 1: Нет доступа к локальным файлам

Веб-агент Claude.ai работает в облаке и не имеет доступа к твоему компьютеру.

**Что видит веб-агент:**
- ❌ Локальные файлы (`/Users/alexander/VK-offee`)
- ❌ Google Drive напрямую
- ✅ Только то, что ты ему отправишь в чате

### Причина 2: Неточное название файла

Ты искал: "Калькуляции кухни"
Реальное название: "Калькулятор для подсчета приготовленных продуктов.csv"

**Решение:** Использовать нечёткий поиск или синонимы.

### Причина 3: Файл не в GitHub (был)

До сегодняшней синхронизации файл был только в Google Drive.

**Сейчас:** ✅ Файл синхронизирован в GitHub

## ✅ Решение: Настроить доступ для всех агентов

### Для локальных агентов (Claude Desktop, Claude Code):

2. **Перезапустить Claude Desktop**

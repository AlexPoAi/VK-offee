## 🎯 Три способа доступа к базе знаний

### 1. Через Telegram-бота ✅ (Готов к запуску)

**Для кого:** Сотрудники, менеджеры, ты сам

**Как работает:**
```
Ты → Telegram-бот → VK-offee (Pack) → AI-ответ
```

**Примеры запросов:**
- "Найди калькуляции кухни"
- "Как приготовить капучино?"
- "Какие поставщики кофе?"
- "Покажи должностную инструкцию бариста"

**Что умеет бот:**
- Ищет в content/ (формализованное знание)
- Ищет в knowledge-base/ (Google Drive документы)
- Генерирует AI-ответы через GPT-4o-mini
- Показывает роли FPF (F1-F9)

**Запуск:**
```bash
cd ~/VK-offee/telegram-bot
./start.sh
```

---

### 2. Через MCP сервер (Model Context Protocol)

**Для кого:** Другие AI-агенты (Claude, GPT, и т.д.)

**Как работает:**
```
Агент Claude → MCP сервер → VK-offee (Pack) → Результат
```

**Что может агент:**
- Читать файлы из content/ и knowledge-base/
- Искать по ключевым словам
- Анализировать структуру SRT (F0-F9)
- Извлекать знания для Downstream

**Пример запроса:**
```
Агент: "Найди калькуляции кухни в VK-offee"

MCP сервер:
1. Ищет в knowledge-base/КУХНЯ/
2. Находит CSV файлы
3. Возвращает содержимое

Результат:
- Калькулятор для подсчета приготовленных продуктов.csv
- Список продуктов _ учёт.csv
- Себестоимость напитков.csv
```

**Настройка MCP:** (если ещё не настроено)
```json
{
  "mcpServers": {
    "vk-offee": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/alexander/VK-offee"]
    }
  }
}
```

### 3. Через GitHub API

**Для кого:** Автоматизация, CI/CD, внешние системы

**Как работает:**
```
Система → GitHub API → VK-offee репозиторий → Файлы
```

**Примеры:**
- Автоматическое обновление документации
- Синхронизация с другими системами
- Мониторинг изменений

**API endpoints:**
```
GET https://api.github.com/repos/alexpoaiagent-sudo/VK-offee/contents/knowledge-base/КУХНЯ
GET https://api.github.com/repos/alexpoaiagent-sudo/VK-offee/contents/content/2.VkusnyCoffeeNetwork
```

## 🔍 Как агент может искать информацию

### Метод 1: Поиск по ключевым словам

```python
# Пример для Python агента
import os
from pathlib import Path

def search_in_pack(keyword):
    pack_path = Path("/Users/alexander/VK-offee")
    results = []

    for file in pack_path.rglob("*.csv"):
        with open(file, 'r', encoding='utf-8') as f:
            content = f.read()
            if keyword.lower() in content.lower():
                results.append(file)

    return results

# Использование
files = search_in_pack("калькул")
# Результат: список файлов с калькуляциями
```

### Метод 2: Поиск по структуре SRT

```python
def find_by_family(family_code):
    """
    Найти документы по семейству F0-F9

    F0 - Метасемейство
    F1-F3 - Надсистема (HoReCa Market)
    F4-F6 - Целевая система (Vkusny Coffee Network)
    F7-F9 - Система создания (Management Team)
    """
    content_path = Path("/Users/alexander/VK-offee/content")

    family_map = {
        "F0": "0.Management",
        "F1": "1.HoReCaMarket",
        "F2": "1.HoReCaMarket",
        "F3": "1.HoReCaMarket",
        "F4": "2.VkusnyCoffeeNetwork",
        "F5": "2.VkusnyCoffeeNetwork",
        "F6": "2.VkusnyCoffeeNetwork",
        "F7": "3.ManagementTeam",
        "F8": "3.ManagementTeam",
        "F9": "3.ManagementTeam",
    }

    folder = family_map.get(family_code)
    if folder:
        return list((content_path / folder).rglob("*.md"))

    return []

# Использование
docs = find_by_family("F8")  # Поставщики и инструменты
```

### Метод 3: Поиск через Telegram-бота API

```python
import requests

def ask_bot(question):
    """Отправить вопрос Telegram-боту"""
    bot_token = os.getenv("TELEGRAM_BOT_TOKEN")
    chat_id = "YOUR_CHAT_ID"

    url = f"https://api.telegram.org/bot{bot_token}/sendMessage"
    data = {
        "chat_id": chat_id,
        "text": question
    }

    response = requests.post(url, json=data)
    return response.json()

# Использование
answer = ask_bot("Найди калькуляции кухни")
```

## ✅ Рекомендации для агентов

### При поиске информации:

1. **Сначала ищи в content/** (формализованное знание)
2. **Потом в knowledge-base/** (рабочие документы)
3. **Используй терминологию из glossary.md**
4. **Учитывай структуру SRT (F0-F9)**

### При извлечении знаний:

1. **Определи семейство** (F0-F9)
2. **Определи систему** (Suprasystem, SoI, Constructor)
3. **Определи роль** (Предприниматель, Инженер, Менеджер)
4. **Проверь согласованность с Pack**

### При формализации:

1. **Добавь YAML frontmatter**
2. **Используй правильную кодировку (UTF-8)**
3. **Добавь в knowledge-inventory.md**
4. **Создай ссылки на связанные документы**

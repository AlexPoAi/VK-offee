## 🧠 Ключевые инсайты

### Про архитектуру FPF+SPF+SRT+FMT:

**FPF (First Principles Framework):**
- Фундамент мышления
- Уже был в `.fpf/` (3.8 МБ)
- Ключевые принципы: Bounded Context, Distinctions, Holons

**SPF (Second Principles Framework):**
- Методология создания Pack
- 12 стадий процесса (00-11)
- Process-lint обязателен перед коммитом
- Hard Gates блокируют некорректные изменения

**SRT (Systems-Roles-Table):**
- Организация знаний через таблицу 3×3
- F0-F9: 10 семейств документов
- Три системы: Suprasystem, System-of-Interest, Constructor
- Три роли: Entrepreneur, Engineer, Manager

**PACK:**
- Формализованное доменное знание
- Структура: 00-manifest, 01-contract, 02-entities, 03-methods, 04-work-products, 05-failure-modes, 06-sota, 07-map
- Шаблоны в `_template/`

### Про различения (Distinctions):

**Критические различения для VK-offee:**
1. Меню ≠ Рецепт
2. Смена ≠ Сотрудник
3. Клиент ≠ Постоянный клиент
4. Процесс ≠ Сервис
5. Стандарт ≠ Навык
6. Калькуляция ≠ Себестоимость
7. Должностная инструкция ≠ Чек-лист

### Про переименование VK-offee → VK-coffee:

**Решение:** НЕ переименовывать сейчас
**Причина:** Может сломать пути, скрипты, Railway, синхронизацию
**Альтернатива:** Использовать "VK-coffee" в документах, "VK-offee" в коде
**Когда можно:** Через 2-4 недели после стабилизации

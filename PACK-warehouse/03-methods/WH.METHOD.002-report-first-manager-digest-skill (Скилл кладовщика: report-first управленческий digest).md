# WH.METHOD.002 — Скилл кладовщика: report-first управленческий digest

## Принцип
Сначала формируем смысл отчёта для руководителя, потом отправляем сообщение в Telegram.

## Входы
1. Карточки цикла (`WH.CARD.*`).
2. Реестр обработки (`WH.REGISTRY.001`).
3. Сводка цикла (`WH.REPORT.002`).
4. Очередь сессий (`WH.SESSION.001`).

## Выходы
1. Полный управленческий отчёт (`WH.WP.*`).
2. Короткий digest в Telegram:
- verdict
- 3 действия
- 1 блок «чего не хватает»
- ссылки на карточки/источники + ссылка на полный отчёт

## Режим доставки
- По умолчанию: только ручной запуск (manual-run), без авто-рассылки по расписанию.
- Авто-рассылка допускается только как явное инженерное исключение.

## Шаблон digest (обязательный)
1. `Вердикт`.
2. `Что сделать сегодня` (3 пункта).
3. `Риски`.
4. `Какие данные запросить`.
5. `Ссылка на полный отчёт`.
6. `Новые карточки (кликабельно)`:
- ссылка на карточку
- ссылка на source-файл

## Quality gate перед отправкой
- [ ] Есть минимум 3 actionable пункта.
- [ ] У каждого пункта есть owner и срок.
- [ ] Нет битых ссылок.
- [ ] Есть блок `data gaps`.
- [ ] Есть статус по ошибкам обработки.

# WH.WP.008 — ABC document chain: live walkthrough

## Зачем

Этот артефакт фиксирует не общую теорию, а живую цепочку обработки документа
`ABC-анализ` в контуре кладовщика:

`документ -> извлечённые поля -> вычисленные метрики -> manager output`

Цель:
- показать, что именно кладовщик реально берёт из `ABC`;
- отделить уже рабочую часть цепочки от хвостов и шума;
- дать эталон для дальнейших document-chain walkthrough по другим типам документов.

## Шаг 3 — Что из этого видит руководитель

### Уже попадает в manager-report

Из `ABC` сейчас в [WH.REPORT.002-warehouse-sync-summary-latest.md](./WH.REPORT.002-warehouse-sync-summary-latest.md) реально попадают:
- `Лидеры ABC`
- `Тяжёлые по себестоимости позиции`
- `Маржа под давлением`
- `Позиции C-категории под пересмотр`

### Уже влияет на decision-layer

`ABC` уже участвует в:
- приоритете части заказов;
- `family-level fallback`, если прямой match между остатками и `ABC` не сработал;
- логике `Не заказывать / сократить` для `C`-позиций.

## Шаг 4 — Что пока ещё не идеально

### Хвосты текущей цепочки

1. В matched data всё ещё попадают строки, которые для руководителя не всегда являются самостоятельным товаром.
Пример:
- `Молоко / 50 гр`

2. В unmatched-preview ещё живут служебные/описательные строки источника:
- `ABC и XYZ анализ`
- `01.04.2026-30.04.2026`
- `Раздел каталога: ...`

3. `ABC` уже даёт сильный manager-signal, но ещё не полностью связан с:
- `sale price`
- `last purchase price`
- `base gross margin`

То есть `ABC -> cost` уже живой,
а `ABC -> полноценная unit economics` ещё нет.

## Truthful verdict

### Что уже можно считать рабочим

- `ABC` больше не просто файл с буквами `A/B/C`.
- Кладовщик реально извлекает из него:
  - продажи
  - выручку
  - себестоимость
  - долю себестоимости
- И превращает это в manager-layer, который уже полезен руководителю.

### Что ещё нельзя считать завершённым

- Нельзя пока говорить, что `ABC` даёт полностью чистую товарную экономику.
- Нужен ещё один bounded layer:
  - отделить полноценные SKU от модификаторов и внутренних полустрок;
  - связать `ABC` с базовой unit economics через закупочную цену и цену продажи.

## Следующий шаг по ABC

1. Ввести фильтр/modifier policy для строк вроде `Молоко / 50 гр`, если это не отдельный товарный объект для руководителя.
2. Связать `ABC` с `каталогом` и `накладными` в один блок:
   - `цена продажи`
   - `последняя закупочная цена`
   - `базовая валовая маржа`
3. После этого сделать вторую live-card:
   - `ABC -> unit economics walkthrough`.

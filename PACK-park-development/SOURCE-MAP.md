# SOURCE-MAP — Парк Голубинка

> Карта физических источников проекта: где лежат оригиналы, сырьё и производные артефакты.

## 1. Верхний контур Pack

- `00-pack-manifest.md` — границы домена
- `CONTEXT.md` — короткий вход в задачу
- `README.md` — стартовая навигация
- `PROJECT-STATUS.md` — оперативная сводка
- `DOCUMENT-REGISTRY.md` — реестр документов
- `COMMUNICATION-REGISTRY.md` — реестр коммуникаций
- `TELEGRAM-TIMELINE.md` — нормализованная лента событий
- `DECISIONS-AND-QUESTIONS.md` — решения и блокеры

## 2. Сырые Telegram-источники

Главный сырой архив:
- `04-work-products (Рабочие продукты)/PARK.WP.005-telegram-export/`

Внутри:
- `result.json` — основной экспорт
- `files/` — вложения из Telegram
- `photos/` — фотографии
- `voice_messages/` — голосовые сообщения
- `video_files/` — видео

Дополнительная JSON-копия:
- `06-documents (Документы парка)/administrative/PARK.WP.005-telegram-correspondence (Переписка Telegram).json`

## 3. Нормализованные коммуникации

Готовые карточки переписок:
- `04-communications (Коммуникации)/PARK.COMM.*`

Их читать раньше, чем сырой экспорт, если задача не требует полного археологического разбора.

## 4. Оригиналы документов

Основное хранилище оригиналов:
- `06-documents (Документы парка)/legal/`
- `06-documents (Документы парка)/administrative/`
- `06-documents (Документы парка)/architectural/`
- `06-documents (Документы парка)/technical/`

Принцип хранения:
- оригиналы `pdf/docx/xlsx/png` храним локально внутри Pack
- в GitHub уходит только навигационный и аналитический слой `md`
- это обеспечивается текущим `.gitignore` в репозитории `VK-offee`, где бинарные файлы игнорируются

Локально зеркально заведён критичный договорной блок ЛУКС:
- `P240`
- `ДС1`
- `ДС2`
- `ДС3`
- `АВР1`
- счета `74/75`
- листы подсчёта площадей `1/2 этаж`

## 5. Markdown-карточки документов

Карточки лежат рядом с оригиналами в тех же подкаталогах `06-documents (...)`.

Принцип:
- оригинал = факт и источник
- карточка = то, что читает агент первым

## 6. Что делать, если пришёл новый документ

1. Положить оригинал в подходящий раздел `06-documents (...)`
2. Создать markdown-карточку `PARK.DOC.*` или `PARK.WP.*`
3. Добавить запись в `DOCUMENT-REGISTRY.md`
4. Если документ возник из переписки — обновить `COMMUNICATION-REGISTRY.md`
5. Если документ меняет ход проекта — обновить `PROJECT-STATUS.md` и `DECISIONS-AND-QUESTIONS.md`
6. Если документ пришёл из `Downloads` или Telegram Desktop — не оставлять его единственным источником вне Pack, а зеркально заносить в Pack

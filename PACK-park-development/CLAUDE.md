## 📋 Ритуал закрытия рабочего продукта

При закрытии **любого** WP по проекту Парк:

### 1. Обновить PROJECT-TIMELINE.md

Добавить запись о событии в хронологию:

```markdown
### YYYY-MM-DD

- **Краткое описание события**
- Что сделано
- Какие документы созданы/обновлены
- Ссылки на PARK.DOC.*, PARK.COMM.*, PARK.WP.*
- Статус проекта (если изменился)
```

### 2. Обновить реестры (если нужно)

- `DOCUMENT-REGISTRY.md` — если создали/обновили документы
- `COMMUNICATION-REGISTRY.md` — если была переписка/встреча
- `PROJECT-STATUS.md` — если изменился статус проекта

### 3. Запустить close-task.sh

```bash
cd ~/Github && bash close-task.sh "PARK.WP: описание что сделано"
```

### 4. Проверить Google Drive

Если создавали документы — убедиться что они в правильной папке:
https://drive.google.com/drive/folders/1-E-pWBQni6Bv2TFAgLOOl3lWMXU9l9lk

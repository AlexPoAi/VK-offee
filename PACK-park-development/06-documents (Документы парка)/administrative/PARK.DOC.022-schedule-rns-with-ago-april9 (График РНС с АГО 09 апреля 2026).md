# PARK.DOC.022 — График РНС с АГО (09 апреля 2026)

## Назначение

PDF-версия графика, присланная ЛУКС `09 апреля 2026` уже после переписки `08 апреля`.
По смыслу это не новый тип документа, а более формальный и пригодный для согласования вид графика, который развивает логику `PARK.DOC.021`.

## Что видно в документе

В графике зафиксированы основные этапы:

- оформление раздела `АР`
- текстовая часть `АР`
- раздел `АР`
- раздел `АГО`
- получение из департамента согласованного `АГО`
- `КР` стадия `П`
- `ПЗ`
- `МОДИ`
- `ПОС`
- `СПОЗУ`
- получение `РНС`
- `КР` стадия `Р`

По читаемым подписям внутри таблицы:

- оформление `АР` — `2 рабочих дня`
- текстовая часть — `14 рабочих дней`
- раздел `АР` — `14 рабочих дней`
- по `АГО` оставлен внешний срок выдачи `30 дней`
- `КР` стадия `П` указана как отдельный следующий блок после согласованного `АГО`

## Точные интервалы по графику

По исходной таймлайн-сетке и длительностям читаются такие интервалы:

| Этап | Интервал | Длительность |
|---|---|---|
| Оформление раздела `АР` | `16.03.2026` → `24.03.2026` | `7 рабочих дней` |
| Текстовая часть раздела `АР` | `20.03.2026` → `08.04.2026` | `14 рабочих дней` |
| Раздел `АГО` | `25.03.2026` → `13.04.2026` | `14 рабочих дней` |
| `КР` стадия `П` + `МОДИ` + `ПОС` + `СПОЗУ` | `09.04.2026` → `07.05.2026` | `20 рабочих дней` |
| `ПЗ` | `24.04.2026` → `13.05.2026` | `14 рабочих дней` |
| `КР` стадия `Р` | `14.05.2026` → `02.06.2026` | `14 рабочих дней` |

Важно:

- этапы подачи пакета на согласование `АГО` и подачи пакета на `РНС` не имеют собственной фиксированной календарной даты внутри документа;
- они описаны как события, от которых зависит запуск следующих этапов.

## Ключевые формулировки

### По АГО

В середине графика сохранена оговорка по внешнему согласованию:

- срок выдачи `АГО` — `30 дней`
- сроки и стоимость последующих этапов могут корректироваться
- основание для корректировки связывается с замечаниями при согласовании `АГО`, выходящими за рамки изменений по окнам, материалам и цвету фасадов

### По старту следующих этапов

В нижнем жёлтом блоке зафиксирована логика:

- начало выполнения подрядчиком следующего этапа — первый рабочий день со дня предоставления заказчиком подрядчику согласованного `АГО`

Это важная формула, потому что она:

- признаёт зависимость подрядчика от внешнего согласования;
- но сама по себе не фиксирует обязательный срок завершения следующих этапов уже **после** получения согласованного `АГО`.

## Аналитический вывод

Этот документ важен тем, что:

1. Переводит график из `.xlsx` в более официальный `pdf`-вид.
2. Подтверждает, что ЛУКС отвечает на требование приложить график к `ДС3`.
3. Не снимает два главных вопроса заказчика:
   - нужны фиксированные сроки следующих этапов после получения согласованного `АГО`;
   - нужно точнее ограничить, какие замечания по `АГО` действительно могут менять сроки и стоимость.
4. График теперь позволяет видеть не только последовательность, но и заявленные календарные интервалы по основным разделам.
5. При этом он всё равно оставляет открытую зону именно там, где проект наиболее чувствителен:
   - между внешним согласованием `АГО` и финальной передачей пакета на `РНС`.

## Связь с предыдущим графиком

По смыслу документ продолжает `PARK.DOC.021`, а не отменяет его.

Главное отличие:

- `PARK.DOC.021` — рабочий файл `.xlsx`
- `PARK.DOC.022` — его формализованная `pdf`-версия, присланная уже в составе нового пакета от `09 апреля`

## Вывод

Документ полезен как входящий артефакт переговоров, но в текущем виде ещё не закрывает позицию заказчика.

Для подтверждения его как приложения к `ДС3` по-прежнему нужны:

- сроковые обязательства после согласованного `АГО`
- более точная граница по замечаниям, меняющим сроки и стоимость

## PACK-cafe-operations

### ЗАКАЗ (обновлено v1.2)
```
DOMAIN: PACK-cafe-operations / ТИП: заказ

1. ОПРЕДЕЛЕНИЕ
   1 заказ клиента, созданный после оплаты

2. СОСТАВ
   - напитки [1..n]
   - порции еды [0..n]

3. СВЯЗИ
   - создаётся из → оплата
   - имеет → оплаты [1..n]
   - связан с → клиент
   - выполняется → сотрудники [1..n]
   - связан с → посадка (опционально)

4. ХАРАКТЕРИСТИКИ
   - время создания
   - время готовности
   - время выдачи

5. СОСТОЯНИЯ
   новый → в работе → готов → выдан

6. ПРАВИЛА
   - заказ не существует без оплаты
   - заказ создаётся после первой оплаты
   - содержит ≥1 позиции
   - может иметь несколько оплат (досплит / доплата)
   - может существовать без посадки (takeaway)

7. ПРИМЕР
   капучино + сэндвич / оплата безнал
```

### ОПЛАТА
```
DOMAIN: PACK-cafe-operations / ТИП: оплата

1. ОПРЕДЕЛЕНИЕ
   1 транзакция денег от клиента

2. СОСТАВ
   нет

3. СВЯЗИ
   - создаёт → заказ
   - относится к → клиент
   - может иметь → возврат

4. ХАРАКТЕРИСТИКИ
   - сумма
   - способ оплаты (наличные / безнал)
   - время
   - комиссия (опционально)

5. СПОСОБЫ ОПЛАТЫ
   - наличные
   - безнал (через терминал)
   (СБП = частный случай безнала, отличается только комиссией)

6. СОСТОЯНИЯ
   выполнена → возвращена (частично / полностью)

7. ПРАВИЛА
   - оплата происходит до создания заказа
   - один заказ может иметь несколько оплат
   - сумма оплат = сумма заказа

8. ПРИМЕР
   500₽, безнал (СБП через терминал)

Различение:
   оплата = движение денег (вход)
   возврат = отдельный объект (движение денег выход)
```

### КЛИЕНТ
```
DOMAIN: PACK-cafe-operations / ТИП: клиент

1. ОПРЕДЕЛЕНИЕ
   1 человек, совершающий оплату и получающий заказ

2. СОСТАВ
   нет

3. СВЯЗИ
   - создаёт → оплата
   - получает → заказ
   - может участвовать в → посадка
   - может иметь → бонусный счёт (в будущем)

4. ХАРАКТЕРИСТИКИ
   Текущее состояние:
   - нет (клиент анонимный)
   В будущем (с лояльностью):
   - имя
   - телефон
   - дата рождения

5. СОСТОЯНИЯ
   нет (клиент существует вне зависимости от идентификации)

6. ПРАВИЛА
   - клиент может быть анонимным
   - клиент может иметь несколько заказов
   - клиент = гость (один тип, не дублируем)

7. РАСШИРЕНИЕ (будущее)
   бонусный счёт → отдельный объект, не часть клиента

8. ПРИМЕР
   Сейчас: анонимный клиент купил кофе
   В будущем: Иван, +7999..., есть бонусный счёт

Различение:
   клиент (человек) ≠ бонусный счёт (баланс)
```

### МЕСТО ХРАНЕНИЯ
```
DOMAIN: PACK-cafe-operations / ТИП: место хранения

1. ОПРЕДЕЛЕНИЕ
   1 физическая зона в точке, где хранятся продукты

2. СОСТАВ
   нет

3. ЭКЗЕМПЛЯРЫ
   - холодильная витрина (десерты, готовая еда)
   - витрина выпечки (выпечка, трубочка с кремом)
   - барный холодильник (ингредиенты: молоко, сиропы)
   - морозилка (замороженные заготовки)

4. СВЯЗИ
   - содержит → продукты (еда / десерты / ингредиенты / заготовки)
   - пополняется → поставка
   - пополняется → перемещение
   - отдаёт → заказ
   - участвует в → списание
   - участвует в → перемещение

5. ХАРАКТЕРИСТИКИ
   - температурный режим (охлаждение / заморозка / комнатная)
   - назначение (еда / выпечка / ингредиенты)

6. ПРАВИЛА
   - каждый продукт имеет одно текущее место хранения
   - перемещение между местами фиксируется
   - списание происходит из конкретного места хранения
   - остатки считаются отдельно по каждому месту

7. ЛОГИКА ДВИЖЕНИЯ
   поставка → место хранения
   морозилка → витрина (разморозка)
   витрина → заказ (продажа)
   витрина → списание (порча)

8. ПРИМЕР
   морозилка: смесь смузи ×10
   холодильная витрина: чизкейк ×4
   витрина выпечки: круассан ×6

Различение (A.7):
   один тип "место хранения" → 4 экземпляра
   различие по физике: температура + функция
```

### ПЕРЕМЕЩЕНИЕ
```
DOMAIN: PACK-cafe-operations / ТИП: перемещение

1. ОПРЕДЕЛЕНИЕ
   1 факт перемещения продукта между местами хранения внутри точки

2. СОСТАВ
   - продукт
   - количество

3. СВЯЗИ
   - из → место хранения
   - в → место хранения

4. ХАРАКТЕРИСТИКИ
   - время

5. СОСТОЯНИЯ
   нет (факт)

6. ПРАВИЛА
   - продукт меняет место хранения
   - перемещение не изменяет общее количество в точке
   - перемещение происходит только внутри точки

7. ПРИМЕР
   смесь смузи ×5: морозилка → витрина (разморозка)

Различение:
   место хранения → где лежит
   перемещение → как меняет место
```

### ОСТАТКИ
```
DOMAIN: PACK-cafe-operations / ТИП: остатки

1. ОПРЕДЕЛЕНИЕ
   количество продукта в конкретном месте хранения на момент времени

2. ДВА ПРЕДСТАВЛЕНИЯ

   А. Расчётные остатки (операционные):
   → характеристика места хранения, не объект
   → формула: поставки − продажи − списания ± перемещения
   → обновляются постоянно, используются в работе

   Б. Фактические остатки (контрольные):
   → объект (через документ инвентаризации)
   → фиксируется вручную, является снимком
   → привязан ко времени

3. СВЯЗЬ МОДЕЛЕЙ
   расчётные остатки
       ↓ сравнение
   фактические остатки
       ↓
   расхождение

4. ПРАВИЛА
   - остатки считаются по каждому месту хранения отдельно
   - перемещения влияют на остатки
   - списания уменьшают остатки
   - продажи уменьшают остатки

5. ПРИМЕР
   витрина: сэндвич ×5 (расчёт)
   инвентаризация: сэндвич ×3 (факт)
   → расхождение: −2

Различение (FPF):
   "остатки как число" → характеристика (НЕ объект)
   "остатки как факт измерения" → объект (через инвентаризацию)
```

### ВОЗВРАТ
```
DOMAIN: PACK-cafe-operations / ТИП: возврат

1. ОПРЕДЕЛЕНИЕ
   операция изменения результата заказа после оплаты

2. СОСТАВ
   нет

3. ТИПЫ
   А. Денежный возврат: деньги возвращаются клиенту
   Б. Замена продукта: деньги не двигаются, меняется продукт

4. СВЯЗИ
   - относится к → заказ
   - может относиться к → оплата (только денежный возврат)
   - связан с → сотрудник

5. ХАРАКТЕРИСТИКИ
   - тип (денежный / замена)
   - сумма (только для денежного)
   - причина
   - время

6. СОСТОЯНИЯ
   нет (факт)

7. ПРАВИЛА
   - возврат не может существовать без заказа
   - денежный возврат ≤ суммы оплаты
   - при замене: старый продукт → списание, новый → создаётся

8. ЛОГИКА
   оплата → заказ
   если ошибка:
     ├── возврат денег (денежный)
     └── замена продукта (без денег)

9. ПРИМЕР
   Денежный: кофе 200₽ → возврат 200₽
   Замена: сделали не тот кофе → старый списали, новый выдали

Различение:
   возврат денег ≠ замена продукта → разные процессы, одна карточка
```

### RETAIL-ТОВАР
```
DOMAIN: PACK-cafe-operations / ТИП: retail-товар

1. ОПРЕДЕЛЕНИЕ
   1 физический товар на полке, продаваемый в точке (кофе зерно, мерч)

2. СОСТАВ
   нет

3. СВЯЗИ
   - входит в → заказ
   - оплачивается через → оплата
   - хранится в → место хранения (полка / склад)

4. ХАРАКТЕРИСТИКИ
   - название
   - цена
   - количество

5. СОСТОЯНИЯ
   нет

6. ПРАВИЛА
   - продаётся через тот же заказ и оплату что напитки и еда
   - участвует в общей выручке смены
   - учитывается в остатках и инвентаризации
   - нет отдельной кассы

7. ПРИМЕР
   заказ: капучино + кофе зерно 250г → одна оплата

Различение:
   напиток / еда / retail → одна логика продажи (A.7)
   retail НЕ выносится в отдельный PACK
```

### РАСХОЖДЕНИЕ
```
DOMAIN: PACK-cafe-operations / ТИП: расхождение

1. ОПРЕДЕЛЕНИЕ
   разница между фактическими и расчётными остатками

2. СОСТАВ
   нет

3. СВЯЗИ
   - инвентаризация (документ)
   - период учёта

4. ХАРАКТЕРИСТИКИ
   - сумма
   - тип: недостача | излишек | списание

5. СОСТОЯНИЯ

### СПИСАНИЕ
```
DOMAIN: PACK-cafe-operations / ТИП: списание

1. ОПРЕДЕЛЕНИЕ
   1 факт удаления позиции (порции еды или ингредиента) из остатков

2. СОСТАВ
   нет

3. СВЯЗИ
   - порция еды / ингредиент (что списано)
   - сотрудник (кто зафиксировал факт)
   - смена

4. ХАРАКТЕРИСТИКИ
   - дата
   - причина: просрочена | повреждена | ошибка хранения
   - количество (для ингредиентов: мл/г; для порций еды: 1 шт)

5. СОСТОЯНИЯ
   нет (факт — не меняется)

6. ПРАВИЛА
   - списание ≠ расхождение
   - списание → уменьшает остатки (учитывается в расчёте)
   - фиксируется кто сделал: ответственность факта, не вины
   - списание порции еды: целиком (1 порция)
   - списание ингредиента: в единицах измерения (мл, г)

7. ПРИМЕР
   сэндвич с курицей / дата: 26.03 / причина: просрочен / сотрудник: Иван
   молоко 500 мл / дата: 26.03 / причина: скисло / сотрудник: Мария
```

### ИНГРЕДИЕНТ (обновлено)
```
DOMAIN: PACK-bar / ТИП: ингредиент

1. ОПРЕДЕЛЕНИЕ
   1 единица сырья, используемого для приготовления напитков на баре

2. СОСТАВ
   нет

3. СВЯЗИ
   - рецепт (входит в состав)
   - напиток (расходуется при приготовлении)
   - поставка (поступает от поставщика)
   - списание (при порче)

4. ХАРАКТЕРИСТИКИ
   - название
   - единица измерения (мл / г / шт)
   - срок годности

5. СОСТОЯНИЯ
   свежий → просрочен
   (логика: если текущее время > срок годности → просрочен → списание)

   - приходит от внешних поставщиков (не с кухни)
   - расходуется в единицах измерения (мл, г)
   - при использовании: уменьшает остатки (плановый расход)
   - при порче: списание (потеря)
   - ингредиент ≠ порция еды (разные единицы учёта)

7. ПРИМЕР
   молоко 3,2% / единица: мл / годен до: 27.03
   сироп лаванда / единица: мл / годен до: 01.09
   альтернативное молоко (овсяное) / единица: мл / годен до: 29.03

Различение:
   ингредиент → сырьё для напитков (учёт в мл/г)
   порция еды → готовый товар с кухни (учёт в штуках)
```

## PACK-management

### ПЕРИОД УЧЁТА
```
DOMAIN: PACK-management / ТИП: период учёта

1. ОПРЕДЕЛЕНИЕ
   интервал времени учёта (~2 недели)

2. СОСТАВ
   - сотрудники (через смены)
   - заказы (ссылки)
   - поставки (ссылки)

3. СВЯЗИ
   - инвентаризация (документ)
   - расхождение (товарное)
   - возмещение

4. ХАРАКТЕРИСТИКИ
   - дата начала
   - дата окончания

5. СОСТОЯНИЯ
   нет

6. ПРАВИЛА
   - заканчивается инвентаризацией

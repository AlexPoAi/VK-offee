# Domain-Contract карточки — все объекты VK-offee

---

## PACK-cafe-operations

### ЗАКАЗ
```
DOMAIN: PACK-cafe-operations / ТИП: заказ

1. ОПРЕДЕЛЕНИЕ
   1 заказ клиента в кофейной точке

2. СОСТАВ
   - напитки [1..n]
   - порции еды [0..n]

3. СВЯЗИ
   - клиент
   - оплаты [1..n]
   - сотрудники [1..n]
   - посадка (опционально)

4. ХАРАКТЕРИСТИКИ
   - время создания
   - время начала
   - время готовности
   - время выдачи

5. СОСТОЯНИЯ
   новый → в работе → готов → выдан

6. ПРАВИЛА
   - содержит ≥1 позиции
   - может иметь несколько оплат
   - может существовать без посадки (takeaway)

7. ПРИМЕР
   кофе + сэндвич / оплата картой
```

### ПОСТАВКА
```
DOMAIN: PACK-cafe-operations / ТИП: поставка

1. ОПРЕДЕЛЕНИЕ
   1 партия еды, поступившая в кофейную точку

2. СОСТАВ
   - порции еды [1..n]

3. СВЯЗИ
   - кухонный цех (источник)
   - кофейная точка (получатель)
   - смена

4. ХАРАКТЕРИСТИКИ
   - дата
   - количество позиций

5. СОСТОЯНИЯ
   нет

6. ПРАВИЛА
   - формируется из заказа кухни
   - пополняет запасы точки (холодильник)

7. ПРИМЕР
   10 сэндвичей, 5 салатов → Тургенева, 15.03
```

### РАСХОЖДЕНИЕ
```
DOMAIN: PACK-cafe-operations / ТИП: расхождение

1. ОПРЕДЕЛЕНИЕ
   разница между фактическими и расчётными остатками

2. СОСТАВ
   нет

3. СВЯЗИ
   - инвентаризация (документ)
   - период учёта

4. ХАРАКТЕРИСТИКИ
   - сумма
   - тип: недостача | излишек | списание

5. СОСТОЯНИЯ
   нет

6. ПРАВИЛА
   - возникает при инвентаризации
   - расхождение = факт − расчёт
   - формула расчёта: остатки = поставки − продажи − списания

7. ПРИМЕР
   −2000₽ недостача по молоку
```

### СПИСАНИЕ
```
DOMAIN: PACK-cafe-operations / ТИП: списание

1. ОПРЕДЕЛЕНИЕ
   1 факт удаления позиции (порции еды или ингредиента) из остатков

2. СОСТАВ
   нет

3. СВЯЗИ
   - порция еды / ингредиент (что списано)
   - сотрудник (кто зафиксировал факт)
   - смена

4. ХАРАКТЕРИСТИКИ
   - дата
   - причина: просрочена | повреждена | ошибка хранения
   - количество (для ингредиентов: мл/г; для порций еды: 1 шт)

5. СОСТОЯНИЯ
   нет (факт — не меняется)

6. ПРАВИЛА
   - списание ≠ расхождение
   - списание → уменьшает остатки (учитывается в расчёте)
   - фиксируется кто сделал: ответственность факта, не вины
   - списание порции еды: целиком (1 порция)
   - списание ингредиента: в единицах измерения (мл, г)

7. ПРИМЕР
   сэндвич с курицей / дата: 26.03 / причина: просрочен / сотрудник: Иван
   молоко 500 мл / дата: 26.03 / причина: скисло / сотрудник: Мария
```

## PACK-kitchen

### ЗАКАЗ КУХНИ
```
DOMAIN: PACK-kitchen / ТИП: заказ кухни

1. ОПРЕДЕЛЕНИЕ
   1 заказ от кофейной точки на производство еды

2. СОСТАВ
   - позиции: порция еды + количество [1..n]

3. СВЯЗИ
   - кофейная точка (заказчик)
   - кухонный цех (исполнитель)
   - поставка (результат)

4. ХАРАКТЕРИСТИКИ
   - дата
   - объём заказа

5. СОСТОЯНИЯ
   принят → в работе → готов → отгружен

6. ПРАВИЛА
   - создаётся точкой
   - выполняется кухней
   - превращается в поставку

7. ПРИМЕР
   Тургенева: салат ×10, сэндвич ×5
```

### ПОРЦИЯ ЕДЫ (обновлено)
```
DOMAIN: PACK-kitchen / ТИП: порция еды

1. ОПРЕДЕЛЕНИЕ
   1 единица готового продукта кухни (в упаковке)

2. СОСТАВ
   - ингредиенты (состав на этикетке)
   - маркировка
       ├── дата производства
       ├── срок годности
       └── время списания

3. СВЯЗИ
   - кухонный цех (создаёт)
   - поставка
   - заказ (точки)

4. ХАРАКТЕРИСТИКИ
   - название
   - срок годности

5. СОСТОЯНИЯ
   свежая → просрочена
   (логика: если текущее время > срок годности → просрочена → списание)

6. ПРАВИЛА
   - создаётся кухней
   - состав фиксируется на этикетке
   - состояние вычисляется по маркировке
   - просроченная → подлежит списанию

7. ПРИМЕР
   Сэндвич с курицей (хлеб, курица, соус)
   Произведён: 15.03 / Годен до: 16.03 12:00

Различение:
   рецепт → «как готовить» (PACK-bar/kitchen)
   состав → «что внутри готового продукта» (этикетка)
```

## PACK-bar

### НАПИТОК
```
DOMAIN: PACK-bar / ТИП: напиток

1. ОПРЕДЕЛЕНИЕ
   1 порция напитка, приготовленная в баре

2. СОСТАВ
   - ингредиенты (через рецепт)

3. СВЯЗИ
   - рецепт
   - заказ
   - оборудование

4. ХАРАКТЕРИСТИКИ
   - объём (30мл / 60мл / 200мл)
   - температура
   - категория (кофе / не кофе / альтернатива)

5. СОСТОЯНИЯ
   готовится → готов

6. ПРАВИЛА
   - создаётся по рецепту
   - входит в заказ

7. ПРИМЕР
   капучино 200 мл, 65°C
```

### ИНГРЕДИЕНТ (обновлено)
```
DOMAIN: PACK-bar / ТИП: ингредиент

1. ОПРЕДЕЛЕНИЕ
   1 единица сырья, используемого для приготовления напитков на баре

2. СОСТАВ
   нет

3. СВЯЗИ
   - рецепт (входит в состав)
   - напиток (расходуется при приготовлении)
   - поставка (поступает от поставщика)
   - списание (при порче)

4. ХАРАКТЕРИСТИКИ
   - название
   - единица измерения (мл / г / шт)
   - срок годности

5. СОСТОЯНИЯ
   свежий → просрочен
   (логика: если текущее время > срок годности → просрочен → списание)

6. ПРАВИЛА
   - приходит от внешних поставщиков (не с кухни)
   - расходуется в единицах измерения (мл, г)
   - при использовании: уменьшает остатки (плановый расход)
   - при порче: списание (потеря)
   - ингредиент ≠ порция еды (разные единицы учёта)

7. ПРИМЕР
   молоко 3,2% / единица: мл / годен до: 27.03
   сироп лаванда / единица: мл / годен до: 01.09
   альтернативное молоко (овсяное) / единица: мл / годен до: 29.03

Различение:
   ингредиент → сырьё для напитков (учёт в мл/г)
   порция еды → готовый товар с кухни (учёт в штуках)
```

## PACK-service

```
DOMAIN: PACK-service / ТИП: стол

1. ОПРЕДЕЛЕНИЕ
   1 физический стол в зоне зала

2. СОСТАВ
   нет

3. СВЯЗИ
   - посадки [0..n]
   - кофейная точка (экземпляр)

4. ХАРАКТЕРИСТИКИ
   - номер

```
DOMAIN: PACK-service / ТИП: посадка

1. ОПРЕДЕЛЕНИЕ
   1 факт использования стола во времени

2. СОСТАВ
   - заказы [0..n]

3. СВЯЗИ
   - стол
   - заказы

4. ХАРАКТЕРИСТИКИ
   - время начала
   - время окончания

5. СОСТОЯНИЯ

7. ПРИМЕР
   стол №3: 10:00–10:40 / 2 заказа

## PACK-management

### ПЕРИОД УЧЁТА
```
DOMAIN: PACK-management / ТИП: период учёта

1. ОПРЕДЕЛЕНИЕ
   интервал времени учёта (~2 недели)

2. СОСТАВ
   - сотрудники (через смены)
   - заказы (ссылки)
   - поставки (ссылки)

3. СВЯЗИ
   - инвентаризация (документ)
   - расхождение
   - возмещение

4. ХАРАКТЕРИСТИКИ
   - дата начала
   - дата окончания

5. СОСТОЯНИЯ
   нет

6. ПРАВИЛА
   - заканчивается инвентаризацией
   - сотрудник входит в период если ≥1 смена

7. ПРИМЕР
   01.03–14.03.2026
```

### ВОЗМЕЩЕНИЕ
```
DOMAIN: PACK-management / ТИП: возмещение

1. ОПРЕДЕЛЕНИЕ
   распределение суммы недостачи между сотрудниками периода

2. СОСТАВ
   - сотрудники [1..n] + доля каждого

3. СВЯЗИ
   - период учёта
   - расхождение

4. ХАРАКТЕРИСТИКИ
   - общая сумма
   - доля на сотрудника = сумма / количество

5. СОСТОЯНИЯ
   нет

6. ПРАВИЛА
   - сумма делится поровну
   - участвуют все сотрудники периода
   - без учёта индивидуальной вины

7. ПРИМЕР
   3000₽ / 3 сотрудника = 1000₽ каждому
```

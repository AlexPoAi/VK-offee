# Шаблон Domain-Contract (жёсткий)

> Использовать для описания КАЖДОГО типа объекта в каждом PACK.

```
DOMAIN: <название PACK>

ТИП: <объект>

---

1. ОПРЕДЕЛЕНИЕ
<что это физически — одно предложение>

---

2. СОСТАВ
<из чего состоит — если контейнер>

---

3. СВЯЗИ
<с какими объектами связан и как>

---

4. ХАРАКТЕРИСТИКИ
<измеримые параметры — A.17: не объекты>

---

5. СОСТОЯНИЯ
<если есть жизненный цикл>

---

6. ПРАВИЛА
<жёсткие инварианты — что ВСЕГДА верно>

---

7. ПРИМЕР
<конкретный кейс из реальной жизни>
```

---

## Примеры заполнения

### Сотрудник
```
DOMAIN: PACK-hr
ТИП: сотрудник

1. ОПРЕДЕЛЕНИЕ
1 человек, выполняющий работу в кофейной точке

2. СОСТАВ
нет

3. СВЯЗИ
- кофейная точка
- смена
- период учёта
- договор
- роли

4. ХАРАКТЕРИСТИКИ
- имя
- дата начала работы

5. СОСТОЯНИЯ
- активен
- уволен

6. ПРАВИЛА
- может иметь несколько ролей
- может участвовать в нескольких сменах

7. ПРИМЕР
Иван, бариста, работает с 01.01.2026
```

### Смена
```
DOMAIN: PACK-management
ТИП: смена

1. ОПРЕДЕЛЕНИЕ
период работы точки или кухни

2. СОСТАВ
- сотрудники
- заказы (для бара)
- поставки (для кухни)

3. СВЯЗИ
- кофейная точка / кухня
- сотрудники
- заказы
- поставки

4. ХАРАКТЕРИСТИКИ
- время начала
- время окончания

5. СОСТОЯНИЯ
нет

6. ПРАВИЛА
- смена ограничена временем
- сотрудник может входить в несколько смен

7. ПРИМЕР
смена 09:00–18:00, Тургенева, 2 сотрудника
```

---

## Проверка (перед сохранением карточки)

- ✅ все типы = объекты
- ✅ методы не смешаны с объектами
- ✅ нет дублирования типов (A.7)
- ✅ время = характеристика (A.17)
- ✅ PACK разделены корректно

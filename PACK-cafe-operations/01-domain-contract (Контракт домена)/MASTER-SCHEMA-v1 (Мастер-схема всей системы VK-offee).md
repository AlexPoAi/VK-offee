# Мастер-схема VK-offee — полная объектная карта

> Разработана 26.03.2026 в ходе FPF-инспекции.
> Основа для написания domain-contract всех PACK.

---

## Схема системы

```
База знаний VK-offee
│
├── PACK-cafe-operations
│   ├── [тип] кофейная точка
│   │     атрибуты: адрес / режим / оборудование / вместимость
│   │     части: бар / зал / витрина / холодильник
│   │     экземпляры: Самокиша / Тургенева / Луговая
│   ├── [тип] заказ
│   ├── [тип] клиент
│   ├── [тип] оплата
│   ├── [тип] поставка
│   ├── [тип] остатки
│   └── [тип] расхождение
│
├── PACK-kitchen
│   ├── [тип] кухонный цех (внутренний / внешний)
│   ├── [тип] порция еды
│   ├── [тип] заготовка
│   └── [тип] заказ кухни
│
├── PACK-bar
│   ├── [тип] напиток
│   ├── [тип] рецепт
│   ├── [тип] ингредиент
│   └── [тип] оборудование
│
├── PACK-service
│   ├── [тип] стол
│   └── [тип] посадка
│
├── PACK-management
│   ├── [тип] смена
│   ├── [тип] период учёта
│   ├── [тип] инвентаризация (документ)
│   ├── [тип] возмещение
│   └── [метрики] потери / точность / % потерь
│
└── PACK-hr
    └── [тип] сотрудник  ← тип здесь, ссылки везде
```

---

## Связи (ядро)

```
кухня → заказ кухни → поставка → точка (витрина/холодильник)

витрина → заказ → оплата

посадка → заказы [0..n]

смена → заказы / поставки

период → инвентаризация → расхождение → возмещение
```

## Правила размещения объектов

| # | Правило |
|---|---------|
| 1 | Объект проходит через несколько поддоменов → живёт в родительском PACK |
| 2 | Объект живёт там, где создаётся как тип |
| 3 | Атрибут на типе — не новый PACK |
| 4 | Не загрязнять объект связями → объект-посредник |
| 5 | Действие → метод (03-methods), состояние → объект/часть |
| 6 | Типы → в предметном PACK, экземпляры → в PACK-cafe-operations |
| 7 | Объект живёт там, где основной цикл его обработки |
| 8 | Связь между доменами — только через объект-посредник |

**Нарушения FPF:**
- A.3 — не путать процесс с объектом (инвентаризация = метод, не объект)
- A.7 — не дублировать описание типа в разных PACK
- A.17 — время = характеристика объекта, не отдельный объект

## Карточки объектов (примеры)

### Заказ
```
Название: Заказ
Тип: объект
PACK: cafe-operations

Описание: 1 заказ клиента

Состав:
  - напитки [1..n]
  - порции еды [0..n]

Связи:
  - клиент
  - оплаты [1..n]
  - сотрудники [1..n]
  - посадка (опционально)

Характеристики:
  - время создания
  - время начала
  - время готовности
  - время выдачи

Состояния:
  новый → в работе → готов → выдан

Пример:
  кофе + сэндвич / оплата картой / посадка стол №3
```

### Поставка
```
Название: Поставка
Тип: объект
PACK: cafe-operations

Описание: 1 партия еды от кухни в точку

Состав:
  - порции еды [1..n]

Связи:
  - кухонный цех (источник)
  - кофейная точка (получатель)
  - смена (кухня и точки)

Характеристики:
  - дата
  - количество позиций

Пример:
  10 сэндвичей, 5 салатов → Тургенева, 15 марта
```
